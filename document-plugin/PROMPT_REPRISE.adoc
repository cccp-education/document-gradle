== Technique de pool de connexion Ollama

`PooledOllamaTranslationAdapter` implémente `TranslationService` (contrat N0 i18n-contracts)
avec un pool de connexion round-robin sur les 29 ports 11437-11465 :

- Chaque port = une instance Ollama Docker avec sa propre Device Key SSH ed25519
- Chaque Device Key = un compte Ollama Pro distinct (17€/mois en annuel)
- Rotation : `AtomicInteger` incrémenté à chaque appel → `index % 29`
- Fallback : si un port échoue (Connection refused, rate-limit 429, timeout) → port suivant
- Tous les ports épuisés → `TranslationResult.Failure`
- `create()` factory : `DEFAULT_PORT_RANGE = 11437..11465`, model = `gemma4:31b-cloud`
- Ports 11434-11436 JAMAIS utilisés (comptes Google personnels, intouchables)
