Relation contenant→contenu, pas parallelisme N2↔N2.

[cols="2,5,5", options="header"]
|===
| | Document (New Orleans) | plantuml-gradle (HTOWN)
| Relation | Contenant (incruste le PNG) | Contenu (genere le PNG)

== Pipeline Livre — Vue d'ensemble

----
[CODEX N2] collectOcr ── photo → LLM → OcrResult.structuredText (AsciiDoc)
                │
                ├─▶ transformChunk → collectIngest → pgvector (RAG)
                │
                └─▶ AsciiDoc source (fichier disque, pont N2↔N2)
                        │
[DOCUMENT N2] ─────────┘
    ├─ generateDocument (optionnel : regenerer/remixer via LLM)
    ├─ enrichDocument (PlantUML via plantuml-gradle, images, includes, passthrough)
    ├─ applyTheme (theme.yml PDF, talaria.css HTML)
    ├─ convertDocumentToHtml → HTML5
    ├─ convertDocumentToPdf  → PDF stagiaire
    └─ convertDocumentToEpub → EPUB3
            │
[RUNNER N3] assembleCompositeContext → deploy gh-pages
----

*Codex gere l'OCR et le RAG. Document gere l'enrichissement et la publication.*
*L'AsciiDoc sur disque est le seul pont entre les deux N2.*
*Runner-gradle N3 orchestre l'ordre d'execution et le deploiement.*
