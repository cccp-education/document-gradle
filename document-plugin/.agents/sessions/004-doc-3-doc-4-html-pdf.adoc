=== DOC-3 — convertDocumentToHtml (AsciidoctorJ html5 backend)

==== DDD ubiquitous language

- `DocumentConverter` — objet domaine convertisseur (lecture seule source → livrable)
- `convertToHtml(source)` — conversion texte (retour String, `standalone(true)` pour doctype)
- `shouldSkipConversion(source, output)` — skip texte (hash lu dans les 3 premières lignes)
- `buildMetadataHeader(source)` — en-tête `<!-- document-gradle — source hash: XXX -->`

==== TDD baby-step (Red → Green)

- **Unit (9)** : DocumentConverterTest
  - doctype présent, sections préservées, code source colorisé, includes résolus
  - source absente lève exception, skip par hash (vrai/faux), metadata header

==== Décision technique

- `Options.builder().standalone(true)` nécessaire pour le `<!DOCTYPE html>` (sinon fragment seul)
- `toFile(false)` pour récupérer le contenu en String (sinon écrit sur disque)

=== DOC-4 — convertDocumentToPdf (AsciidoctorJ pdf backend)

==== DDD ubiquitous language — formats binaires

- `convertToPdf(source, output)` — délègue à `convertToFile` (écrit directement dans un fichier)
- `convertToFile(source, backend, output)` — backend binaire générique (pdf, epub3)
- `shouldSkipBinaryConversion(source, output)` — skip binaire (fichier sidecar `.sourcehash`)
- `writeBinaryMetadataHeader(source, output)` — écrit le hash dans `<output>.sourcehash`

==== Distinction texte vs binaire

[cols="2,5,5"]
|===
| | Format texte (HTML, DocBook, ManPage) | Format binaire (PDF, EPUB)

| Conversion | `convert()` retourne String | `convertToFile()` écrit dans File
| Hash source | Inséré dans le fichier (commentaire HTML) | Fichier sidecar `.sourcehash`
| Skip | `shouldSkipConversion` (lit 3 lignes) | `shouldSkipBinaryConversion` (lit sidecar)
| Metadata | `buildMetadataHeader` | `writeBinaryMetadataHeader`
|===

== Décisions techniques

. **standalone(true)** : requis pour le `<!DOCTYPE html>` complet (header + footer).
   Sans cette option, AsciidoctorJ produit un fragment HTML sans doctype.
. **toFile(false)** : récupère le contenu HTML en String. Pour PDF, `toFile(output)`
   écrit directement le binaire dans le fichier (pas de retour String possible).
. **Sidecar `.sourcehash`** : le PDF étant binaire, le hash source ne peut pas
   être inséré dans le fichier. Un fichier sidecar `<output>.sourcehash` stocke
   le hash pour l'invalidation (économice d'encre). Pattern DDD distinct du HTML.
. **asciidoctorj-pdf 2.3.23** : module séparé (non inclus dans asciidoctorj core).
   Dernière stable compatible avec AsciidoctorJ 3.0.0.
. **convertBinary méthode privée** : factorise skip + conversion + metadata
   pour les formats binaires (PDF maintenant, EPUB en DOC-5).
