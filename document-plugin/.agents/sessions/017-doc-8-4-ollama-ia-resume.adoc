== Décisions techniques

=== Décorateur sur AsciidocReleaseNotesRenderer

`OllamaAsciidocReleaseNotesRenderer` ne re-implemente pas le template
AsciiDoc — il délègue à `AsciidocReleaseNotesRenderer` et insère la section
`== Résumé` entre le titre (`= Release Notes {version}`) et le corps. La
séparation title/body est faite par `indexOf("\n\n")` (le titre est la
première ligne, suivie d'une ligne vide). Fallback = rendu pur sans résumé.

=== runBlocking pour bridge sync N0 / suspend LLM

Le contrat N0 `ReleaseNotesRenderer.render()` est synchrone (interface
Java-friendly). `DocumentLlmProvider.call()` est `suspend` (coroutine Kotlin).
Le bridge utilise `runBlocking { provider.call(prompt) }` — pattern déjà
utilisé dans `DocumentGenerationGraph.kt` (DOC-2). Pas de fuite de contexte
IO car la task Gradle n'est pas structurée concurrency-wise.

=== Fallback gracieux (Loi de l'Économie d'Encre)

`runCatching { runBlocking { provider.call(...) } }.getOrNull() ?: return base`
attrape toute exception (Ollama down, timeout, réseau). Pas de retry, pas de
crash du build Gradle. Le release notes sans résumé reste un release notes
valide. Le résumé est une valeur ajoutée, pas une dépendance dure.

=== Factory configDrivenWithLlm sépare les responsabilités

`configDriven(projectDir, parser)` (DOC-8.2) reste pour les renderers sans
LLM (asciidoc/markdown/json). `configDrivenWithLlm(projectDir, parser, provider)`
est l'extension pour `ollama-asciidoc`. Le provider n'est instancié que si
`rendererType == "ollama-asciidoc"` dans la task — pas de coût d'init LLM
pour les autres renderers.

=== DSL llmMode mirror de llmMode extension (DOC-2)

Le mode `"fake"` est partagé avec `DocumentExtension.llmMode` (DOC-2
generateDocument). Même factory `DocumentLlmProviderFactory`, même
sémantique (fake = tests sans réseau, ollama = production local).
