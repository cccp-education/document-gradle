== Décisions techniques

=== @Input String plutôt que @InputDirectory

Le directory `build/release-notes/` n'existe pas toujours (pipeline sans
release notes). `@InputDirectory` + `@Optional` lève une erreur de validation
Gradle si le directory n'existe pas. Solution : `@Input Property<String>` qui
porte le path absolu, résolu en `File` dans le task action avec garde
`takeIf { it.exists() && it.isDirectory }`.

=== Premier entry = release notes primaire

Le tableau `releaseNotes` peut contenir plusieurs fichiers (asciidoc + markdown
par exemple). Le `releaseNotesPath` dans metadata.json porte le premier entry
uniquement — c'est le "release notes primaire" que runner-gradle déploie en
priorité pour gh-pages.

=== JsonInclude.Include.NON_NULL

`DocumentMetadata` sérialise désormais avec `JsonInclude.Include.NON_NULL` —
les champs null (`releaseNotesPath`, `releaseNotesRenderer`) sont omis du JSON
plutôt que sérialisés en `null`. Garde le metadata.json propre quand le
pipeline n'a pas produit de release notes.
