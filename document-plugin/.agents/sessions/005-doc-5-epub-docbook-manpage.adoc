== Décisions techniques

. **asciidoctorj-epub3 2.2.0** : module séparé (non inclus dans asciidoctorj core).
   Dernière stable compatible avec AsciidoctorJ 3.0.0. L'EPUB est un zip (signature PK\x03\x04).
. **DocBook5 natif** : backend `docbook5` inclus dans asciidoctorj core, aucune extension requise.
   Produit du XML avec namespace `http://docbook.org/ns/docbook`.
. **ManPage natif** : backend `manpage` inclus dans asciidoctorj core.
   La source AsciiDoc doit avoir `:doctype: manpage` + section NAME + SYNOPSIS.
   Produit du troff (directives .TH, .SH, .ds).
. **EPUB binaire (sidecar)** : comme PDF, l'EPUB étant binaire, le hash source est stocké
   dans un fichier sidecar `.sourcehash`. `convertBinary` factorise skip + conversion + metadata.
. **DocBook/ManPage texte (header)** : comme HTML, le hash est inséré dans le fichier
   de sortie via `buildMetadataHeader`. `shouldSkipConversion` lit les 3 premières lignes.
. **convertBinary refactor** : paramètre `epicId` retiré (plus de stub), signature simplifiée
   à `(docSource, output, logger, backend)`. Factorise PDF + EPUB.

== Architecture — 5 formats complets

[cols="2,2,2,3"]
|===
| Format | Backend | Type | Skip

| HTML | html5 | texte | header metadata (3 lignes)
| PDF | pdf | binaire | sidecar .sourcehash
| EPUB | epub3 | binaire | sidecar .sourcehash
| DocBook | docbook5 | texte | header metadata (3 lignes)
| ManPage | manpage | texte | header metadata (3 lignes)
|===
